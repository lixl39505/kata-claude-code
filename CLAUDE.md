# Claude Code 开发约束

- 只将 specs 中的文档作为单一事实来源
- app 和 lib 用于存放代码, 需要再去搜索

## 前置要求

在修改任何代码前，必须：

1. 阅读 `specs/requirements.md` —— 了解业务需求
2. 阅读 `specs/architecture.md` —— 了解系统设计
3. 输出变更计划，包括：
   - 修改目标
   - 变更文件列表
   - 影响范围
   - 回滚方案

## 修改约束

- 不得进行与当前任务无关的重构
- 不得实现未定义功能
- 不得引入新的后端框架
- 不得引入新的 ORM 框架
- 不得引入新的数据库系统
- 不得引入额外验证库（统一使用 Zod）
- 所有业务功能必须具备单元测试, 必须覆盖：
  - 正常路径
  - 关键失败路径

## 质量门禁

所有修改都必须通过以下测试命令：

- lint
- typecheck
- test

禁止跳过测试直接提交.
