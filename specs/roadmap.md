# Roadmap（能力地图）

## Phase 1 · Core Foundation（当前阶段）

- 用户认证（登录 / 注册 / session）
- Project 管理（创建 / 列表 / 详情）
- Issue 基础能力（创建 / 列表 / 详情）
- Issue 状态流转
- Issue 审计日志
- Issue 评论
- Issue 指派
- Issue 查询与筛选

目标：形成一个“单用户可用”的完整 Issue 管理闭环

---

## Phase 2 · Collaboration（协作能力）

- Project 成员管理（加入 / 移除）
- 基础角色模型（owner / member）
- Issue 指派边界（只能指派成员）
- 评论 @ 提及（基础版）
- 通知基础（最小可用）

目标：从单人系统 → 多人协作系统

---

## Phase 3 · Productivity（效率提升）

- Issue 批量操作（批量状态 / 指派）
- 高级筛选（多条件组合 + 保存）
- 自定义视图（如 backlog / my issues）
- 基础统计（数量 / 状态分布）

目标：从“能用” → “高效使用”

---

## Phase 4 · Hardening（工程化与稳定性）

- 数据一致性（事务优化）
- 错误恢复机制
- 基础安全加固
- 性能优化（查询优化 / 索引）
- 日志与监控

目标：从“产品” → “可长期运行的系统”
